DDON – Server Essentials

Core server components for Dragon’s Dogma Online

Overview

DDON (Dragon’s Dogma Online) is a fan-driven server project aimed at restoring the original online experience by re-implementing the game’s backend services.

Purpose

This page documents the essential server-side components required to run a DDON private server. It serves as a technical reference for developers and contributors.

Server Architecture

Client
  ↓
Auth Server
  ↓
Game Server
  ↓
Database

Core Components

Auth Server

Handles account authentication, session management and server selection.

Game Server

Manages gameplay logic such as characters, quests, NPCs, combat and instances.

Database

Stores all persistent data including accounts, characters, items and progress.

Network / Packet Layer

Responsible for DDON-specific client-server communication and packet handling.

Basic Startup Flow

  1. Configure database connection
  2. Start Auth Server
  3. Start Game Server
  4. Connect with DDON client

Project Status

Development is currently Work in Progress. Many systems may be incomplete or experimental.
